Getting Started

Erste Schritte

Analysiere dein SwiftUI-Projekt in 3 einfachen Schritten

1

Ordner öffnen

Drücke ⌘ O, um deinen Projektordner auszuwählen. Canopy erkennt automatisch .swift-Dateien und beginnt mit der AST-Analyse.

2

View-Baum erkunden

Das linke Panel zeigt deine Dateiliste, in der Mitte wird der View-Baum angezeigt. Klicke auf einen Knoten, um den zugehörigen Quellcode im rechten Panel hervorgehoben zu sehen.

3

Qualität analysieren

Das untere Panel zeigt Qualitätswerte, Abhängigkeitsgraphen und Statistiken. Klicke auf ein Noten-Badge, um die Komplexitätsaufschlüsselung zu sehen.

TIPCanopy ist nur lesend. Es verändert oder löscht niemals den Code deines Projekts. Nutze es mit vollem Vertrauen.
Features

Hauptfunktionen

Von Canopy bereitgestellte Inspektionsfunktionen

🌲

View-Baum-Anzeige

Visualisiert SwiftUI View-Hierarchien als Baumstruktur. Erkennt verschachtelte Views, @ViewBuilder und View-Funktionen in extensions.

📊

Qualitätsanalyse

Vergibt Noten von A bis F für jede View. Bewertet Komplexität, Verschachtelungstiefe, State-Anzahl, Body Weight und Modifier-Reihenfolge.

🔗

Abhängigkeitsgraph

Visualisiert Abhängigkeiten zwischen Views. Erkennt zirkuläre Referenzen, findet isolierte Views und verfolgt den State-zu-Binding-Fluss.

🔍

State-Audit

Erkennt @State-, @Binding-, @Environment- und @Observable-Properties. Warnt bei fehlenden private-Modifiern und übermäßiger State-Nutzung.

🎨

Syntax-Hervorhebung

Farblich codierte Swift-Codeanzeige. Hebt Zeilen hervor, die zu View-Knoten gehören. Klick auf Code führt zur Rückwärtssuche im Baum.

🔨

Xcode-Integration

Klicke auf das Hammer-Symbol, um die entsprechende Zeile direkt in Xcode zu öffnen (via xed --line).

📐

Architektur-Regeln

Definiere eigene Regeln mit .canopy-rules.json. Teile Namenskonventionen, State-Obergrenzen und Verschachtelungstiefen-Beschränkungen mit deinem Team.

🧩

Plugins

Füge eigene Regeln, Exporter und Analyzer als Plugins hinzu. Durchsuche den Plugin-Katalog und installiere von dort.

Interface

Panel-Layout

Layout mit 3 Panes + unterem Panel

📁

Links: Dateiliste

Liste der analysierten Swift-Dateien. Jede Datei zeigt ein Noten-Badge. Klick zum Auswählen.

🌿

Mitte: View-Baum

View-Hierarchiestruktur. Zeigt Breadcrumbs, Peek Body (body-Vorschau) und Code Vision-Metriken.

📝

Rechts: Quellcode

Zeilennummerierte Codeanzeige. Ziehe Zeilennummern, um Bereiche auszuwählen, dann mit ⌘C kopieren. Klick auf Code führt zur Rückwärtssuche im Baum.

📈

Unten: Analyse-Panel

Drei Tabs: Qualität, Abhängigkeiten, Statistik. Größe anpassbar und einklappbar. Klick auf Noten-Badges öffnet Detail-Popover.

CLI

Kommandozeile

Headless-Ausführung für die Integration in CI/CD-Pipelines

# Analyseergebnisse im JSON-Format ausgeben
$ Canopy --analyze /path/to/project --format json

# SARIF-Format (für GitHub Code Scanning)
$ Canopy --analyze /path/to/project --format sarif

# JSON Schema anzeigen
$ Canopy --format json-schema
SETUPBevor du die CLI verwendest, öffne den Projektordner einmal in Canopy (GUI). Die Zugriffsrechte werden gespeichert, danach kannst du von der CLI aus analysieren.
CILade die SARIF-Ausgabe zu GitHub Actions hoch, um Qualitätswarnungen direkt in Pull Requests anzuzeigen.
