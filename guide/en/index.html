Getting Started

Getting Started

Analyze your SwiftUI project in 3 simple steps

1

Open a Folder

Press ⌘ O to select your project folder. Canopy automatically detects .swift files and begins AST analysis.

2

Explore the View Tree

The left panel shows your file list, the center displays the View tree. Click a node to see its source code highlighted in the right panel.

3

Analyze Quality

The bottom panel shows quality scores, dependency graphs, and statistics. Click a grade badge to see the complexity breakdown.

TIPCanopy is read-only. It never modifies or deletes any of your project's code. Use it with confidence.
Features

Key Features

Inspection capabilities provided by Canopy

🌲

View Tree Display

Visualize SwiftUI View hierarchies as a tree structure. Detects nested Views, @ViewBuilder, and View functions in extensions.

📊

Quality Analysis

Assigns A through F grades to each View. Evaluates complexity, nesting depth, State count, body weight, and modifier order.

🔗

Dependency Graph

Visualize dependencies between Views. Detect circular references, find isolated Views, and trace State-to-Binding flow.

🔍

State Audit

Detects @State, @Binding, @Environment, and @Observable properties. Warns about missing private modifiers and excessive State usage.

🎨

Syntax Highlighting

Color-coded Swift code display. Highlights lines corresponding to View nodes. Click code to reverse-lookup in the tree.

🔨

Xcode Integration

Click the hammer icon to open the corresponding line directly in Xcode (via xed --line).

📐

Architecture Rules

Define custom rules with .canopy-rules.json. Share naming conventions, State limits, and nesting depth restrictions across your team.

🧩

Plugins

Add custom rules, exporters, and analyzers as plugins. Browse and install from the plugin catalog.

Interface

Panel Layout

3 panes + bottom panel layout

📁

Left: File List

List of analyzed Swift files. Each file shows a grade badge. Click to select.

🌿

Center: View Tree

View hierarchy structure. Shows breadcrumbs, Peek Body (body preview), and Code Vision metrics.

📝

Right: Source Code

Line-numbered code display. Drag line numbers to select ranges, then copy with ⌘C. Click code to reverse-lookup in the tree.

📈

Bottom: Analysis Panel

Three tabs: Quality, Dependencies, Statistics. Resizable and collapsible. Click grade badges for detail popovers.

CLI

Command Line

Headless execution for local automation

# Output analysis results in JSON format
$ Canopy --analyze /path/to/project --format json

# SARIF format
$ Canopy --analyze /path/to/project --format sarif

# Display JSON Schema
$ Canopy --format json-schema
SETUPBefore using the CLI, open the project folder in Canopy (GUI) once. Access permissions are saved, and you can analyze from the CLI from then on.
PRE-COMMITAdd it to a pre-commit hook to catch F-grade code before committing.
