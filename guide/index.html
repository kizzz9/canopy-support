Getting Started

はじめに

3ステップで SwiftUI プロジェクトを解析できます

1

フォルダを開く

⌘ O でプロジェクトフォルダを選択します。Canopy が .swift ファイルを自動検出し、AST 解析を開始します。

2

View ツリーを探索

左パネルにファイル一覧、中央に View ツリーが表示されます。ノードをクリックすると、右パネルにソースコードがハイライト表示されます。

3

品質を分析

下部パネルで品質スコア・依存グラフ・統計情報を確認できます。各 View のグレードバッジをクリックすると、複雑度の内訳が表示されます。

Tip

Canopy は読み取り専用です。プロジェクトのコードを変更・削除することは一切ありません。安心して使えます。

Features

主な機能

Canopy が提供するインスペクション機能

🌲

View ツリー表示

SwiftUI の View 階層をツリー構造で可視化。ネストされた View、@ViewBuilder、extension 内の View 関数も検出します。

📊

品質分析

各 View に A〜F のグレードを付与。複雑度・ネスト深度・State 数・Body Weight・Modifier 順序を評価します。

🔗

依存グラフ

View 間の依存関係を可視化。循環参照の検出、孤立 View の発見、State→Binding フローの追跡ができます。

🔍

State 監査

@State / @Binding / @Environment / @Observable のプロパティを検出し、private 修飾子の欠落や過剰な State を警告します。

🎨

シンタックスハイライト

Swift コードを色分け表示。View ノードに対応する行をハイライトし、コードクリックでツリーを逆引きします。

🔨

Xcode 連携

ハンマーアイコンをクリックすると、対応する行で Xcode を直接開きます（xed --line）。

📐

アーキテクチャルール

.canopy-rules.json でカスタムルールを定義。命名規則・State 上限・ネスト深度制限などをチームで共有できます。

🧩

プラグイン

カスタムルール・エクスポーター・アナライザーをプラグインとして追加可能。カタログから選んでインストールできます。

Interface

パネル操作

3ペイン + ボトムパネルの構成

📁

左: ファイルリスト

解析対象の Swift ファイル一覧。各ファイルにグレードバッジが付きます。クリックで選択。

🌿

中央: View ツリー

View の階層構造。パンくずリスト、Peek Body（body プレビュー）、Code Vision メトリクスを表示。

📝

右: ソースコード

行番号付きコード表示。行ドラッグで範囲選択 → ⌘C でコピー。コードクリックでツリー逆引き。

📈

下: 分析パネル

品質・依存・統計の3タブ切替。リサイズ・折りたたみ可能。グレードバッジクリックで内訳ポップオーバー。

Legend

アイコン・バッジ凡例

ツリーとファイル一覧に表示される各要素の意味

ノードアイコンの色

レイアウト — VStack / HStack / ZStack ● 青
インタラクション — Button / Toggle / Picker ● 緑
メディア — Image ● 紫
ナビゲーション — NavigationStack / TabView ● オレンジ
リスト系 — List / ForEach / ScrollView ● シアン
条件分岐 — if / switch ● 黄
区切り — Divider / Spacer ● グレー

バッジ・マーカー

[H] [V] [Z] [S] [L] [F] [G] — レイアウトバッジ HStack / VStack / ZStack / ScrollView / List / Form / Grid
3 — Modifier 数バッジ そのノードの .modifier の数
👁 — Peek Body 右クリックで body を展開プレビュー
L42 — 行番号 ソースコード上の行位置
NOW — 1時間以内に更新 ファイル一覧に表示
TODAY — 24時間以内に更新 ファイル一覧に表示
A〜F — グレードバッジ クリックで複雑度の内訳を表示

CLI

コマンドライン

CI/CD パイプラインに組み込めるヘッドレス実行

# JSON 形式で解析結果を出力
$ Canopy --analyze /path/to/project --format json

# SARIF 形式（GitHub Code Scanning 連携）
$ Canopy --analyze /path/to/project --format sarif

# JSON Schema を表示
$ Canopy --format json-schema
CI 連携

SARIF 出力を GitHub Actions にアップロードすると、PR 上で品質警告がインライン表示されます。
