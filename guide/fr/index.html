Getting Started

Prise en main

Analysez votre projet SwiftUI en 3 étapes simples

1

Ouvrir un dossier

Appuyez sur ⌘ O pour sélectionner le dossier de votre projet. Canopy détecte automatiquement les fichiers .swift et lance l'analyse AST.

2

Explorer l'arborescence des Views

Le panneau de gauche affiche la liste des fichiers, le panneau central affiche l'arborescence des Views. Cliquez sur un nœud pour voir son code source mis en évidence dans le panneau de droite.

3

Analyser la qualité

Le panneau inférieur affiche les scores de qualité, les graphes de dépendances et les statistiques. Cliquez sur un badge de grade pour voir le détail de la complexité.

TIPCanopy est en lecture seule. Il ne modifie ni ne supprime jamais le code de votre projet. Utilisez-le en toute confiance.
Features

Fonctionnalités clés

Les capacités d'inspection offertes par Canopy

🌲

Affichage de l'arborescence des Views

Visualisez les hiérarchies de Views SwiftUI sous forme d'arborescence. Détecte les Views imbriquées, les @ViewBuilder et les fonctions de View dans les extensions.

📊

Analyse de qualité

Attribue un grade de A à F à chaque View. Évalue la complexité, la profondeur d'imbrication, le nombre de State, le Body Weight et l'ordre des modifiers.

🔗

Graphe de dépendances

Visualisez les dépendances entre les Views. Détectez les références circulaires, repérez les Views isolées et suivez le flux State→Binding.

🔍

Audit State

Détecte les propriétés @State, @Binding, @Environment et @Observable. Avertit en cas d'absence du modifier private ou d'utilisation excessive de State.

🎨

Coloration syntaxique

Affichage du code Swift en couleurs. Met en évidence les lignes correspondant aux nœuds de View. Cliquez sur le code pour effectuer une recherche inversée dans l'arborescence.

🔨

Intégration Xcode

Cliquez sur l'icône de marteau pour ouvrir directement la ligne correspondante dans Xcode (via xed --line).

📐

Règles d'architecture

Définissez des règles personnalisées avec .canopy-rules.json. Partagez les conventions de nommage, les limites de State et les restrictions de profondeur d'imbrication au sein de votre équipe.

🧩

Plugins

Ajoutez des règles, exporteurs et analyseurs personnalisés sous forme de plugins. Parcourez et installez-les depuis le catalogue de plugins.

Interface

Disposition des panneaux

Disposition en 3 volets + panneau inférieur

📁

Gauche : liste des fichiers

Liste des fichiers Swift analysés. Chaque fichier affiche un badge de grade. Cliquez pour sélectionner.

🌿

Centre : arborescence des Views

Structure hiérarchique des Views. Affiche le fil d'Ariane, Peek Body (aperçu du body) et les métriques Code Vision.

📝

Droite : code source

Affichage du code avec numéros de ligne. Faites glisser les numéros de ligne pour sélectionner une plage, puis copiez avec ⌘C. Cliquez sur le code pour effectuer une recherche inversée dans l'arborescence.

📈

Bas : panneau d'analyse

Trois onglets : Qualité, Dépendances, Statistiques. Redimensionnable et repliable. Cliquez sur les badges de grade pour afficher le détail dans une fenêtre contextuelle.

CLI

Ligne de commande

Exécution headless pour l'intégration dans un pipeline CI/CD

# Affiche les résultats d'analyse au format JSON
$ Canopy --analyze /path/to/project --format json

# Format SARIF (pour GitHub Code Scanning)
$ Canopy --analyze /path/to/project --format sarif

# Afficher le schéma JSON
$ Canopy --format json-schema
SETUPAvant d'utiliser la CLI, ouvrez d'abord une fois le dossier du projet dans Canopy (interface graphique). Les droits d'accès sont enregistrés, et vous pourrez ensuite analyser depuis la CLI.
CITéléversez la sortie SARIF vers GitHub Actions pour afficher les avertissements de qualité directement dans les pull requests.
