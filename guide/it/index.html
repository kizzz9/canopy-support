Getting Started

Per iniziare

Analizza il tuo progetto SwiftUI in 3 semplici passaggi

1

Apri una cartella

Premi ⌘ O per selezionare la cartella del tuo progetto. Canopy rileva automaticamente i file .swift e avvia l'analisi AST.

2

Esplora l'albero delle View

Il pannello sinistro mostra l'elenco dei file, il centro mostra l'albero delle View. Clicca su un nodo per vedere il codice sorgente evidenziato nel pannello destro.

3

Analizza la qualità

Il pannello inferiore mostra i voti di qualità, i grafi delle dipendenze e le statistiche. Clicca su un badge del voto per vedere la ripartizione della complessità.

TIPCanopy è di sola lettura. Non modifica né elimina mai il codice del tuo progetto. Usalo con fiducia.
Features

Funzionalità principali

Le funzionalità di ispezione offerte da Canopy

🌲

Visualizzazione dell'albero delle View

Visualizza le gerarchie di View SwiftUI come struttura ad albero. Rileva View annidate, @ViewBuilder e funzioni View nelle extension.

📊

Analisi della qualità

Assegna un voto da A a F a ogni View. Valuta complessità, profondità di annidamento, numero di State, Body Weight e ordine dei modifier.

🔗

Grafo delle dipendenze

Visualizza le dipendenze tra le View. Rileva riferimenti circolari, individua le View isolate e traccia il flusso da State a Binding.

🔍

Audit @State

Rileva le proprietà @State, @Binding, @Environment e @Observable. Avvisa in caso di modificatori private mancanti e uso eccessivo di State.

🎨

Evidenziazione della sintassi

Visualizzazione a colori del codice Swift. Evidenzia le righe corrispondenti ai nodi View. Clicca sul codice per la ricerca inversa nell'albero.

🔨

Integrazione con Xcode

Clicca sull'icona del martello per aprire direttamente la riga corrispondente in Xcode (tramite xed --line).

📐

Regole di architettura

Definisci regole personalizzate con .canopy-rules.json. Condividi convenzioni di denominazione, limiti di State e restrizioni sulla profondità di annidamento con il tuo team.

🧩

Plugin

Aggiungi regole personalizzate, esportatori e analizzatori come plugin. Sfoglia e installa dal catalogo dei plugin.

Interface

Disposizione dei pannelli

Struttura a 3 riquadri + pannello inferiore

📁

Sinistra: elenco file

Elenco dei file Swift analizzati. Ogni file mostra un badge del voto. Clicca per selezionare.

🌿

Centro: albero delle View

Struttura della gerarchia delle View. Mostra breadcrumb, Peek Body (anteprima del body) e metriche Code Vision.

📝

Destra: codice sorgente

Visualizzazione del codice con numeri di riga. Trascina i numeri di riga per selezionare un intervallo, poi copia con ⌘C. Clicca sul codice per la ricerca inversa nell'albero.

📈

In basso: pannello di analisi

Tre schede: Qualità, Dipendenze, Statistiche. Ridimensionabile e comprimibile. Clicca sui badge dei voti per i popover di dettaglio.

CLI

Riga di comando

Esecuzione headless per l'integrazione con pipeline CI/CD

# Restituisce i risultati dell'analisi in formato JSON
$ Canopy --analyze /path/to/project --format json

# Formato SARIF (per GitHub Code Scanning)
$ Canopy --analyze /path/to/project --format sarif

# Mostra lo schema JSON
$ Canopy --format json-schema
SETUPPrima di usare la CLI, apri una volta la cartella del progetto in Canopy (GUI). I permessi di accesso verranno salvati, e da quel momento potrai analizzare dalla CLI.
CICarica l'output SARIF su GitHub Actions per mostrare gli avvisi di qualità direttamente nelle pull request.
