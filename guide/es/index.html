Getting Started

Primeros pasos

Analiza tu proyecto SwiftUI en 3 sencillos pasos

1

Abre una carpeta

Pulsa ⌘ O para seleccionar la carpeta de tu proyecto. Canopy detecta automáticamente los archivos .swift y comienza el análisis AST.

2

Explora el árbol de Views

El panel izquierdo muestra la lista de archivos, y en el centro se muestra el árbol de Views. Haz clic en un nodo para ver su código fuente resaltado en el panel derecho.

3

Analiza la calidad

El panel inferior muestra las puntuaciones de calidad, los grafos de dependencias y las estadísticas. Haz clic en una insignia de grado para ver el desglose de complejidad.

TIPCanopy es de solo lectura. Nunca modifica ni elimina el código de tu proyecto. Puedes usarlo con total confianza.
Features

Funciones principales

Capacidades de inspección que ofrece Canopy

🌲

Árbol de Views

Visualiza la jerarquía de Views de SwiftUI como una estructura de árbol. Detecta Views anidadas, @ViewBuilder y funciones View en extensions.

📊

Análisis de calidad

Asigna a cada View una nota de A a F. Evalúa la complejidad, la profundidad de anidamiento, el número de State, el Body Weight y el orden de modifiers.

🔗

Grafo de dependencias

Visualiza las dependencias entre Views. Detecta referencias circulares, encuentra Views huérfanas y rastrea el flujo de State a Binding.

🔍

Auditoría de State

Detecta las propiedades @State, @Binding, @Environment y @Observable. Advierte sobre la falta de modifiers private y el uso excesivo de State.

🎨

Resaltado de sintaxis

Muestra el código Swift con colores. Resalta las líneas correspondientes a los nodos View. Haz clic en el código para hacer una búsqueda inversa en el árbol.

🔨

Integración con Xcode

Haz clic en el icono de martillo para abrir directamente la línea correspondiente en Xcode (mediante xed --line).

📐

Reglas de arquitectura

Define reglas personalizadas con .canopy-rules.json. Comparte con tu equipo las convenciones de nomenclatura, los límites de State y las restricciones de profundidad de anidamiento.

🧩

Plugins

Añade reglas, exportadores y analizadores personalizados como plugins. Explora e instala desde el catálogo de plugins.

Interface

Distribución de paneles

Diseño de 3 paneles + panel inferior

📁

Izquierda: Lista de archivos

Lista de los archivos Swift analizados. Cada archivo muestra una insignia de grado. Haz clic para seleccionarlo.

🌿

Centro: Árbol de Views

Estructura jerárquica de Views. Muestra la ruta de navegación, Peek Body (vista previa del body) y las métricas de Code Vision.

📝

Derecha: Código fuente

Código mostrado con números de línea. Arrastra los números de línea para seleccionar rangos y cópialos con ⌘C. Haz clic en el código para hacer una búsqueda inversa en el árbol.

📈

Inferior: Panel de análisis

Tres pestañas: Calidad, Dependencias, Estadísticas. Redimensionable y contraíble. Haz clic en las insignias de grado para ver ventanas emergentes con los detalles.

CLI

Línea de comandos

Ejecución headless para integrarlo en pipelines de CI/CD

# Genera los resultados del análisis en formato JSON
$ Canopy --analyze /path/to/project --format json

# Formato SARIF (para GitHub Code Scanning)
$ Canopy --analyze /path/to/project --format sarif

# Muestra el JSON Schema
$ Canopy --format json-schema
SETUPPara analizar un proyecto con la CLI, ábrelo primero una vez en Canopy (GUI). El permiso de acceso se guardará y a partir de entonces podrás analizarlo desde la CLI.
CISube la salida SARIF a GitHub Actions para mostrar las advertencias de calidad directamente en los pull requests.
